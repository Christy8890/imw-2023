Poster Process

By: Christy Wang

1. Brainstorming

The first step of my process is reviewing the project brief and brainstorming based on the given prompts on Figma. The prompt I chose was “what’s something you daydream about”, I thought about some non-fictional and fun fictional story ideas. The one I went ahead with was growing tomatoes! I thought it was fun to bring users to the journey of growing vegetables.

2. Visual Research

During Visual research, I went to Pinterest to grab some simple instructional images to get some inspiration. At first I misunderstood the assignment briefly and thought we could use images, but after consulting with Peter, I realised I had to look for more simpler shapes and elements Im able to code purely with CSS.

After doing some digging, I was very inspired by very simple, but vibrant character shapes. I loved ones with cute smiley faces on fruits, since I was doing tomatoes I thought that would be perfect.

3. Lo-Fi Wireframes I

Next, I designed a set of UI elements to base off of when I code the storybook. This includes some potential buttons I want to use, illustrations and typography.

4. Lo-Fi Wireframes II

Then, I went ahead with storyboarding the flow of the web pages and adding in the copy. I went with a more linear storytelling method.

5. UI Elements

Next I explored different ways I would want to lay out my story. However, after consulting with Peter, we both agree that it was lacking a bit of interactive-ness. There was only one page and it was only scrollable and clickable. So I went back to redo my wireframes.

6. Hi-Fi Wireframes Attempt I

My 2nd draft consisted of 5 pages of scrollable and clickable buttons that brought users to the next and previous pages.

7. Hi-Fi Wireframes Attempt II

After that I went straight to the high-fidelity wireframes, laying out all the UI elements and exploring different types of design choices.

8. Hi-Fi Wireframes Attempt III

Then, I tested out the design as an interactive prototype to take a look at the navigation.

9. Hi-Fi Wireframes Attempt IV

Finally, I began coding. This was done on a test html file. There were lots of errors with my code such as spacing issues and random spacing at the bottom. It wasn’t processing as I was hoping.

10. Coding Attempt II

After the test coding, I went back to Figma and made some changes to the ending and made all the toto clickable. I took out the last page as I felt that it was not that important. I refocused my time to making the storybook more interactive.

11. Coding Attempt III

The first new interaction I added was a hover effect to the totos. When you hover over them, it gives the impresseion that it grew!

12. Coding Attempt IV

The 2nd new interaction I implemented was making all the totos clickable. When you select the right one it brings you to the new page and if you select the wrong one, it will take u to this "try again" page. I like how I gamified the storybook a bit to make it more fun!
